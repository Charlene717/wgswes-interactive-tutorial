Adapter Trimming

移除 adapter 與低品質 base——但別 over-trim：軟剪裁與硬剪裁的權衡。

Remove adapters and low-quality bases — but don't over-trim. Soft vs hard trimming trade-offs.

概覽

什麼時候要 trim？什麼時候不要？

Adapter trimming 處理兩種問題：(1) insert size 短於 read length 時 3' 端讀到 adapter；(2) 3' 端品質下降。但是過度 trim 會降低比對率、丟失資訊。

爭議：MacManes (2014, Front. Genet.) 與 Del Fabbro (2013) 都指出，Q5-Q10 以下的 aggressive quality trimming 反而降低下游表現。對於 BWA-MEM2 等 soft-clipping 的 aligner，輕度 trim 即可；BBTools 作者 Bushnell 主張幾乎不需 quality trim。但對於精確要求極高的 (panel/MRD)、或舊式 aligner，trim 仍然有效。

Adapter trimming handles two problems: (1) insert size shorter than read length reading into adapter at the 3' end; (2) 3' end quality drop. But over-trimming reduces mapping rate and discards information.

The debate: MacManes (2014, Front. Genet.) and Del Fabbro (2013) showed aggressive quality trimming below Q5–Q10 actually hurts downstream performance. For soft-clipping aligners like BWA-MEM2, light trimming suffices; BBTools author Bushnell argues quality trimming is mostly unnecessary. For panels/MRD or legacy aligners, trimming still matters.

💡
2026 默認方案：用 fastp 的 --detect_adapter_for_pe，搭配 -q 15 -l 50（Q15 滑動窗、最小長度 50 bp）。BWA-MEM2 接住其餘的軟剪裁。 2026 default: use fastp --detect_adapter_for_pe with -q 15 -l 50 (Q15 sliding window, min length 50 bp). BWA-MEM2 handles the rest via soft-clipping.

程式碼

實作範例

# fastp — recommended default
fastp \
    -i fq/S1_R1.fq.gz -I fq/S1_R2.fq.gz \
    -o trim/S1_R1.fq.gz -O trim/S1_R2.fq.gz \
    --detect_adapter_for_pe \
    --qualified_quality_phred 15 \
    --length_required 50 \
    --cut_tail --cut_tail_window_size 4 --cut_tail_mean_quality 15 \
    -j qc/fastp/S1.json -h qc/fastp/S1.html \
    --thread 8

# UMI support (e.g. Twist UMI panels)
fastp ... --umi --umi_loc per_read --umi_len 8 ...
# Trimmomatic — paired-end with Illumina TruSeq adapters
trimmomatic PE -threads 8 -phred33 \
    fq/S1_R1.fq.gz fq/S1_R2.fq.gz \
    trim/S1_R1.fq.gz trim/S1_R1_unpaired.fq.gz \
    trim/S1_R2.fq.gz trim/S1_R2_unpaired.fq.gz \
    ILLUMINACLIP:TruSeq3-PE-2.fa:2:30:10:2:keepBothReads \
    LEADING:3 TRAILING:3 SLIDINGWINDOW:4:15 MINLEN:50
# BBDuk — strict k-mer adapter removal
bbduk.sh \
    in1=fq/S1_R1.fq.gz in2=fq/S1_R2.fq.gz \
    out1=trim/S1_R1.fq.gz out2=trim/S1_R2.fq.gz \
    ref=adapters,artifacts,phix \
    ktrim=r k=23 mink=11 hdist=1 tbo tpe \
    qtrim=r trimq=15 minlength=50 \
    threads=8
⚠️
UMI 警告：若使用 UMI panel，必須先抽出 UMI 再 trim，否則 UMI 序列會被當 adapter 剪掉。fastp 的 --umi_loc 可指定 UMI 位置 (read1/read2/per_index/per_read)。 UMI warning: for UMI panels you must extract UMI before trimming, else UMI bases get clipped. fastp --umi_loc specifies UMI location (read1/read2/per_index/per_read).
