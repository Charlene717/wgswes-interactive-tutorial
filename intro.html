WGS / WES 概論

在進入分析流程前，先弄清楚 WGS、WES、Panel 在覆蓋率、成本與臨床用途上的根本差異。

Before diving into the pipeline — understand the fundamental differences between WGS, WES, and Panel sequencing in coverage, cost, and clinical use.

概覽

為什麼要學 WGS / WES？

全基因組定序 (WGS, Whole Genome Sequencing) 與 全外顯子定序 (WES, Whole Exome Sequencing) 是現代基因組醫學的基石。從罕見疾病診斷、癌症精準治療、藥物基因組學、產前篩檢、到族群遺傳學，幾乎所有需要鑑定 SNV / indel / CNV / SV 的研究都建立在這套流程上。

截至 2026 年，WGS 30× 的市場成本已降至 USD $200 以下 (Illumina NovaSeq X、Ultima Genomics UG100、MGI DNBSEQ-T7)，PacBio Revio 與 ONT PromethION 也讓 long-read 普及。但「從 FASTQ 到臨床等級變異報告」中間隔著數十個工具與決策——這正是本教學要教的。

Whole Genome Sequencing (WGS) and Whole Exome Sequencing (WES) are the cornerstones of modern genomic medicine. From rare-disease diagnosis to precision oncology, pharmacogenomics, prenatal screening, and population genetics — virtually all work that requires identifying SNVs / indels / CNVs / SVs is built on this pipeline.

As of 2026, 30× WGS market cost has dropped below USD $200 (Illumina NovaSeq X, Ultima Genomics UG100, MGI DNBSEQ-T7). PacBio Revio and ONT PromethION have made long-read accessible. But between "FASTQ" and a clinical-grade variant report sit dozens of tools and decisions — that's what this tutorial covers.

💡
核心原則：沒有「最好的流程」——只有「最適合你的樣本、預算、目標的流程」。GATK Best Practices、DRAGEN、DeepVariant、nf-core/sarek 都是合理選擇。本教學以 GATK 4.x + Snakemake/Nextflow 為主軸，但會在每章比較替代方案。 Core principle: there is no "best pipeline" — only one best suited to your sample, budget, and goals. GATK Best Practices, DRAGEN, DeepVariant, and nf-core/sarek are all reasonable choices. This tutorial centres on GATK 4.x + Snakemake/Nextflow, but compares alternatives in every chapter.

🌳 平台選擇決策流程

Q1:
需要偵測 CNV / SV / 結構變異 / 非編碼區變異？ → 是 → 選 WGS 30×（必要時加 long-read）。
Q2:
僅關注編碼區、預算非常緊？ → 是 → WES 100× 仍合理；但比較整體成本後 WGS 常勝出。
Q3:
需要偵測 ctDNA 或低 VAF 體細胞變異 (<1%)？ → 是 → Targeted panel 1,000–10,000× + UMI。
Q4:
SV-rich 疾病 (杜興肌肉失養、亨丁頓舞蹈)？ → 是 → 考慮 PacBio HiFi 或 ONT long-read，T2T-CHM13 作為參考。
Q1:
Need CNV / SV / non-coding variants? → Yes → Choose WGS 30× (consider adding long-read).
Q2:
Coding only, tight budget? → Yes → WES 100× still reasonable — but compare total cost; WGS often wins.
Q3:
ctDNA or low-VAF somatic (<1%)? → Yes → Targeted panel 1,000–10,000× + UMI.
Q4:
SV-rich disease (DMD, Huntington)? → Yes → PacBio HiFi or ONT long-read; consider T2T-CHM13.

程式碼

範例：從 FASTQ 到 VCF 的最小可行流程

# GATK Best Practices 4.x 最小流程 / Minimal pipeline
# 1. Alignment
bwa-mem2 mem -t 16 -R "@RG\tID:S1\tSM:S1\tPL:ILLUMINA\tLB:lib1" \
    ref.fa S1_R1.fq.gz S1_R2.fq.gz | samtools sort -@ 4 -o S1.bam -

# 2. Mark duplicates
gatk MarkDuplicatesSpark -I S1.bam -O S1.dedup.bam -M S1.metrics.txt

# 3. BQSR (Base Quality Score Recalibration)
gatk BaseRecalibrator -I S1.dedup.bam -R ref.fa \
    --known-sites dbsnp.vcf.gz --known-sites Mills_and_1000G.vcf.gz -O recal.table
gatk ApplyBQSR -I S1.dedup.bam -R ref.fa --bqsr-recal-file recal.table -O S1.recal.bam

# 4. HaplotypeCaller (gVCF mode)
gatk HaplotypeCaller -I S1.recal.bam -R ref.fa -O S1.g.vcf.gz -ERC GVCF

# 5. GenotypeGVCFs (single sample — for multi-sample use GenomicsDBImport)
gatk GenotypeGVCFs -V S1.g.vcf.gz -R ref.fa -O S1.vcf.gz

# 6. Hard filter (for single sample or small cohorts)
gatk VariantFiltration -V S1.vcf.gz -R ref.fa -O S1.filtered.vcf.gz \
    --filter-expression "QD < 2.0" --filter-name QD2 \
    --filter-expression "FS > 60.0" --filter-name FS60 \
    --filter-expression "MQ < 40.0" --filter-name MQ40
# Snakemake 範例 — 多樣本平行化 / Multi-sample parallel
SAMPLES = ["S1", "S2", "S3"]

rule all:
    input: expand("vcf/{s}.filtered.vcf.gz", s=SAMPLES)

rule align:
    input: r1="fq/{s}_R1.fq.gz", r2="fq/{s}_R2.fq.gz"
    output: "bam/{s}.bam"
    threads: 16
    shell:
        "bwa-mem2 mem -t {threads} -R '@RG\\tID:{wildcards.s}\\tSM:{wildcards.s}\\tPL:ILLUMINA' "
        "ref.fa {input.r1} {input.r2} | samtools sort -@ 4 -o {output} -"

rule call:
    input: "bam/{s}.recal.bam"
    output: "gvcf/{s}.g.vcf.gz"
    shell: "gatk HaplotypeCaller -I {input} -R ref.fa -O {output} -ERC GVCF"
# nf-core/sarek — production-grade pipeline / 生產級流程
nextflow run nf-core/sarek \
    -profile docker \
    --input samplesheet.csv \
    --genome GATK.GRCh38 \
    --tools haplotypecaller,deepvariant,strelka,mutect2,manta,cnvkit,vep \
    --joint_germline true \
    --outdir results/

# samplesheet.csv:
# patient,sample,lane,fastq_1,fastq_2
# P1,S1,L001,fq/S1_R1.fq.gz,fq/S1_R2.fq.gz
