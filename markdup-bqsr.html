STEP 5 / 16

MarkDup 與 BQSR

去除 PCR/光學重複、校正鹼基品質——以及為什麼 DRAGEN 流程可以跳過 BQSR。

Remove PCR/optical duplicates and recalibrate base qualities — and why DRAGEN can skip BQSR.

概覽

為什麼要做 MarkDup 與 BQSR？

比對完成後，BAM 中存在兩個必須處理的偏差：(1) 重複 reads——PCR 或光學原因產生的「同一片段被測到多次」；(2) 鹼基品質偏差——廠商給的 Phred Q 並非真實錯誤率，存在系統性偏移。兩者若不處理，會直接造成假陽性變異。

標準順序：BWA-MEM2 → sort → MarkDuplicates → BQSR → variant calling。MarkDup 必須在 coordinate-sorted BAM 上跑；BQSR 必須在 MarkDup 之後 (否則重複 reads 會扭曲錯誤模型)。

After alignment, the BAM contains two biases that must be addressed: (1) duplicate reads — PCR or optical duplicates of the same fragment; (2) base-quality bias — vendor Phred Q is not the true error rate. Untreated, both inflate false positives.

Standard order: BWA-MEM2 → sort → MarkDuplicates → BQSR → variant calling. MarkDup needs coordinate-sorted BAM; BQSR must follow MarkDup (or duplicate reads skew the error model).

💡
DRAGEN-GATK 功能等價：Illumina DRAGEN 引入了 hardware-aware 模型，可省略 BQSR 並仍與 GATK 結果相容 (Illumina/Broad DRAGEN-GATK best practices 2021)。但若使用 BWA-MEM2 + GATK，仍建議跑 BQSR。 DRAGEN-GATK functional equivalence: Illumina DRAGEN's hardware-aware model lets you skip BQSR while staying GATK-compatible (DRAGEN-GATK best practices 2021). For BWA-MEM2 + GATK pipelines, keep BQSR on.
重複類型

一、PCR vs Optical duplicates

🧪

PCR duplicates

library prep 時 PCR 過度放大同一原始分子。判別：同一 5' 對位起點且 insert size 相同。低 input DNA / 高 PCR cycles 是主因。

Library PCR over-amplifying the same original molecule. Detect: identical 5' coordinate and insert size. Driven by low input or excessive PCR cycles.

💡

Optical duplicates

NovaSeq patterned flowcell 中相鄰 well 互染。判別：同 tile、像素距離 < 2500。比例高需檢查 cluster density。

Adjacent wells on NovaSeq patterned flowcell cross-contaminate. Detect: same tile, pixel distance < 2500. High rate flags over-clustering.

⚠️
NovaSeq 提醒：patterned flowcell 的 optical duplicate 比例 (常達 5–15%) 遠高於舊式 HiSeq。Picard MarkDuplicates 預設 OPTICAL_DUPLICATE_PIXEL_DISTANCE=100，對 NovaSeq 應設為 2500。 NovaSeq note: optical duplicate fraction on patterned flowcells (often 5–15%) far exceeds legacy HiSeq. Picard default OPTICAL_DUPLICATE_PIXEL_DISTANCE=100 should be raised to 2500 for NovaSeq.

BQSR 原理

三、BQSR：用已知變異點校正錯誤

BQSR (Base Quality Score Recalibration) 的核心假設：所有「非已知變異」的鹼基不一致，都是測序錯誤。GATK 使用 dbSNP + Mills_and_1000G_gold_standard.indels.vcf 作為已知位點，把這些位點 mask 掉，剩下的 mismatch 計算經驗錯誤率，再回頭校正報告的 Q 值。

四個 covariates：ReadGroup、ReportedQuality、Cycle (機台週期)、Context (前兩個鹼基)。每組組合給出 (reported Q, empirical Q)，建表後 ApplyBQSR 改寫 BAM 的 Q 欄位。

BQSR (Base Quality Score Recalibration) assumes: any mismatch outside known variant sites is a sequencing error. GATK masks dbSNP + Mills_and_1000G_gold_standard.indels.vcf; the remaining mismatches give an empirical error rate that corrects the reported Q.

Four covariates: ReadGroup, ReportedQuality, Cycle, Context (two preceding bases). Each combination yields (reported Q, empirical Q); ApplyBQSR rewrites Q fields in the BAM.

互動模擬

Reported Q vs Empirical Q

調整 reported Q 與校正偏移 (delta)，觀察 BQSR 校正後的曲線。理想情況下，校正後 reported = empirical (沿對角線)。

Tune reported Q and recalibration offset to see the BQSR-corrected curve. Ideally, after correction reported = empirical (along the diagonal).

四、什麼時候可以跳過 BQSR？

🌳 BQSR 必要性

Q1:
使用 DRAGEN 流程？ → 是 → 可跳過 BQSR (DRAGEN-GATK 等價)。
Q2:
panel 深度 >1000×？ → 是 → BQSR 邊際效益小，可省略。
Q3:
非人類物種 (無 dbSNP 等已知位點)？ → 自行用高信度變異 bootstrap 或跳過。
Q4:
標準 30× WGS + GATK HaplotypeCaller？ → 必須跑 BQSR。
Q1:
Running DRAGEN? → Yes → Skip BQSR (DRAGEN-GATK equivalent).
Q2:
Panel depth >1000×? → Yes → Marginal benefit; safe to omit.
Q3:
Non-human species (no known sites)? → Bootstrap with high-confidence calls or skip.
Q4:
Standard 30× WGS + HaplotypeCaller? → Run BQSR.
程式碼

MarkDup + BQSR 實作

# Picard MarkDuplicates — NovaSeq pixel distance / NovaSeq 像素距離
gatk MarkDuplicates \
    -I bam/S1.sorted.bam \
    -O bam/S1.md.bam \
    -M qc/S1.metrics.txt \
    --OPTICAL_DUPLICATE_PIXEL_DISTANCE 2500 \
    --CREATE_INDEX true
# BQSR: build table → apply
gatk BaseRecalibrator \
    -I bam/S1.md.bam -R ref/GRCh38.fa \
    --known-sites resources/dbsnp.vcf.gz \
    --known-sites resources/Mills_and_1000G_gold_standard.indels.vcf.gz \
    -O bqsr/S1.recal.table

gatk ApplyBQSR \
    -I bam/S1.md.bam -R ref/GRCh38.fa \
    --bqsr-recal-file bqsr/S1.recal.table \
    -O bam/S1.recal.bam

# Optional QC: before/after table comparison
gatk AnalyzeCovariates \
    -before bqsr/S1.recal.table -after bqsr/S1.recal.after.table \
    -plots qc/S1.bqsr.pdf
# DRAGEN-GATK equivalent: skip BQSR
dragen \
    --ref-dir ref/GRCh38_dragen/ \
    --fastq-list fastq_list.csv \
    --output-directory out/ \
    --output-file-prefix S1 \
    --enable-map-align true \
    --enable-duplicate-marking true \
    --enable-variant-caller true \
    --vc-emit-ref-confidence GVCF
