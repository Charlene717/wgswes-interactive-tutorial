序列比對 (BWA-MEM2)

將 reads 對位到參考基因組——BWA-MEM2 的 SIMD 加速、CRAM 壓縮、Read Group 與 GRCh38/T2T 抉擇。

Map reads to the reference — BWA-MEM2 SIMD speedup, CRAM compression, Read Groups, and GRCh38 vs T2T.

概覽

為什麼比對是 WGS/WES 的關鍵步驟？

比對 (alignment) 的目的是找出每條 read 最可能來自的基因組位置。錯誤的對位 → 錯誤的變異呼叫。短讀長 (~150bp) 在重複區、segmental duplication、HLA、KIR 等位點極具挑戰；長讀則改變遊戲規則但成本與錯誤模式不同。

2026 主流：BWA-MEM2 (Vasimuddin 2019, IPDPS) 是 BWA-MEM (Li 2013, arXiv) 的 SIMD 加速重寫，輸出與原版位元等價，但速度快 2–3.5 倍。記憶體需求較大 (索引 ~80 GB for GRCh38)，需 AVX2/AVX512 CPU。

Alignment finds each read's most likely genomic origin. Wrong mapping → wrong variant calls. Short reads (~150 bp) struggle in repeats, segmental duplications, HLA, KIR. Long reads change the game but with different cost and error profiles.

2026 default: BWA-MEM2 (Vasimuddin 2019, IPDPS) — a SIMD rewrite of BWA-MEM (Li 2013, arXiv) producing bit-identical output 2–3.5× faster. Larger memory footprint (~80 GB index for GRCh38) and AVX2/AVX512 CPU required.

💡
核心鏈：BWA-MEM2 (短讀比對) → samtools sort (依座標排序) → samtools index (建立 .bai/.csi) → 進入 MarkDuplicates。CRAM 比 BAM 省 30–50% 空間，但讀寫需參考。 Core chain: BWA-MEM2 (short-read align) → samtools sort → samtools index (.bai/.csi) → MarkDuplicates. CRAM saves 30–50% over BAM but requires the reference for I/O.

二、SAM / BAM / CRAM 與 Read Groups

📄

SAM

純文字版本，可讀但體積大；正式 pipeline 不常用，多用於除錯。

Plain text — readable but huge; rarely used in production, mainly for debugging.

📦

BAM

BGZF 壓縮二進位 SAM；GATK 預設 I/O。需 .bai 索引。

BGZF-compressed binary SAM; GATK default I/O. Requires .bai index.

🗜️

CRAM

參考導向壓縮，省 30–50%；歐洲 EGA 主流。讀寫需參考。

Reference-based compression, 30–50% smaller; EGA default. Needs reference for I/O.

Read Group (@RG) 是 GATK 必填的元資料：ID 唯一識別、SM 樣本名、PL 平台 (ILLUMINA)、LB library、PU platform unit (flowcell.lane.barcode)。MarkDuplicates 依 RG 區分 library；BQSR 依 RG 學習錯誤模型。RG 錯了，後面全錯。

Read Group (@RG) is mandatory metadata for GATK: ID unique, SM sample, PL platform (ILLUMINA), LB library, PU platform unit (flowcell.lane.barcode). MarkDuplicates uses RG to separate libraries; BQSR learns the error model per RG. Wrong RG → everything downstream is wrong.

⚠️
CIGAR 中 S 與 H 的差別：soft-clip (S) 保留序列但不對位 (位元仍在 BAM 中)；hard-clip (H) 將序列從 BAM 直接移除 (節省空間，但無法回溯)。BWA-MEM2 預設輸出 soft-clip，supplementary alignment 才會出現 H。 CIGAR S vs H: soft-clip (S) keeps the sequence but excludes it from alignment (bases still in BAM); hard-clip (H) removes bases entirely (space-saving but irreversible). BWA-MEM2 emits S by default; H appears only on supplementary alignments.
參考基因組

三、GRCh38 ALT/decoy vs T2T-CHM13

GRCh38 + ALT-aware：BWA 的 .alt 索引讓 reads 同時打到 primary 與 ALT contigs，並透過 bwa-postalt.js 解決多對位問題。decoy (hs38d1) 收集已知混雜序列，吸收原本誤對位到人類基因組的 reads。臨床流程目前仍主用 GRCh38。

T2T-CHM13 v2.0 (Nurk 2022, Science) 補齊了著絲粒、rDNA、衛星序列，研究級項目 (尤其 segmental duplication、satellite) 越來越多採用；但臨床參考座標仍以 GRCh38 為主，需 liftover 工具如 levioSAM2。

GRCh38 + ALT-aware: BWA's .alt index lets reads map to both primary and ALT contigs; bwa-postalt.js resolves the duplicates. Decoy (hs38d1) absorbs known contaminants that would otherwise mis-map. Clinical pipelines still use GRCh38.

T2T-CHM13 v2.0 (Nurk 2022, Science) closes centromeres, rDNA, and satellites; research projects (esp. segmental duplications, satellites) increasingly adopt it. Clinical coordinates remain GRCh38 — use levioSAM2 for liftover.

互動模擬

Read length × insert size → fragment overlap

調整讀長與 insert size，觀察 PE reads 是否重疊 (overlap)、是否會讀進 adapter。insert < 2×L 時 reads 互相重疊；insert < L 時 3' 端會讀到 adapter。

Tune read length and insert size to see whether PE reads overlap or read into adapter. When insert < 2×L they overlap; when insert < L the 3' end reads into adapter.

平台是 Illumina 短讀 (PE 150)？ → 是 → BWA-MEM2 (預設) 或 DRAGEN (HW 可用時)。
Q2:
PacBio HiFi 或 ONT 長讀？ → minimap2 with -x map-hifi 或 -x map-ont。
Q3:

Q4:

Q1:
Illumina short read (PE 150)? → Yes → BWA-MEM2 (default) or DRAGEN if available.
Q2:
PacBio HiFi or ONT? → minimap2 with -x map-hifi / -x map-ont.
Q3:
Need ALT contig handling (HLA, KIR, IGH)? → Use GRCh38 + .alt index + bwa-postalt.js.
Q4:
Studying segmental duplications / centromeres? → Consider T2T-CHM13, liftover back to GRCh38.
程式碼

比對與排序

# BWA-MEM2 + sort + index, with proper @RG / BWA-MEM2 加排序與索引
bwa-mem2 index -p ref/GRCh38 ref/GRCh38.fa

RG="@RG\tID:S1.L001\tSM:S1\tPL:ILLUMINA\tLB:S1.lib1\tPU:HXXXXDSXY.1.ATCACG"

bwa-mem2 mem -t 16 -R "$RG" -K 100000000 -Y \
    ref/GRCh38 trim/S1_R1.fq.gz trim/S1_R2.fq.gz \
  | samtools sort -@ 8 -m 2G -o bam/S1.sorted.bam -
samtools index -@ 4 bam/S1.sorted.bam
# minimap2 for PacBio HiFi or ONT
minimap2 -t 16 -ax map-hifi ref/GRCh38.fa hifi/S1.fq.gz \
  | samtools sort -@ 8 -o bam/S1.hifi.bam -
samtools index bam/S1.hifi.bam

# ONT R10.4.1
minimap2 -t 16 -ax map-ont ref/GRCh38.fa ont/S1.fq.gz | samtools sort -o bam/S1.ont.bam -
# Convert BAM → CRAM (saves 30–50% disk)
samtools view -@ 8 -C -T ref/GRCh38.fa \
    -o cram/S1.cram bam/S1.sorted.bam
samtools index cram/S1.cram

# Read back to BAM if a tool needs it
samtools view -@ 8 -b -T ref/GRCh38.fa -o bam/S1.bam cram/S1.cram
