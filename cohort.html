Trio / 大規模佇列分析

從家族 trio de novo 偵測，到 UK Biobank、All of Us、gnomAD v4 等百萬級佇列的 burden test 與 GWAS。

From family-trio de novo detection to million-scale cohort burden tests and GWAS (UK Biobank, All of Us, gnomAD v4).

概覽

Trio 與 Cohort：兩種尺度的問題

Trio (3 人 family) 是罕病診斷的黃金標準：父、母、proband 同時定序，可直接識別 de novo 變異（每代約 60-80 個 SNV de novo）並驗證遺傳模式。

大型 cohort 把焦點轉為「基因—疾病關聯」：gnomAD v4 (Karczewski 2024 Nature) 807,162 exomes + 76,215 genomes；UK Biobank 500k WGS (2023-2024 完整釋出)；All of Us 規劃 100 萬人（2024 釋出 ~245k WGS）。

分析框架：Hail (Spark-based)、Project Glow、Bigstack 可橫向處理 PB 級資料。

Trio (3-person family) is the gold standard for rare-disease diagnosis: father, mother, and proband sequenced together — directly identifies de novo variants (~60–80 SNV de novo per generation) and validates inheritance modes.

Large cohorts shift focus to "gene-disease association": gnomAD v4 (Karczewski 2024 Nature) 807,162 exomes + 76,215 genomes; UK Biobank 500k WGS (released 2023–2024); All of Us targeting 1M (245k WGS released 2024).

Frameworks: Hail (Spark-based), Project Glow, Bigstack scale to PB-level data.

程式碼

PED、GATK trio、Hail

# trio.ped — 6 columns, tab-separated
FAM01  father    0          0          1  1
FAM01  mother    0          0          2  1
FAM01  proband   father     mother     1  2

# GATK posterior + de novo filter
gatk CalculateGenotypePosteriors \
    -V joint.vcf.gz -ped trio.ped \
    --supporting-callsets gnomad-af.vcf.gz \
    -O trio.posterior.vcf.gz

gatk VariantFiltration -V trio.posterior.vcf.gz \
    --genotype-filter-expression "GQ < 20" --genotype-filter-name lowGQ \
    -O trio.filt.vcf.gz

gatk VariantAnnotator -V trio.filt.vcf.gz -A PossibleDeNovo \
    -ped trio.ped -O trio.deNovo.vcf.gz
# Hail — load gnomAD-scale MatrixTable, query
import hail as hl
hl.init()
mt = hl.read_matrix_table('gs://gnomad-public/release/4.0/ht/exomes.mt')

# Filter by rarity + LoF
rare_lof = mt.filter_rows(
    (mt.freq[0].AF < 1e-5) &
    (mt.vep.most_severe_consequence == 'stop_gained'))

# Per-gene aggregation
per_gene = rare_lof.group_rows_by(rare_lof.vep.gene_symbol)\
    .aggregate(n=hl.agg.count())
# REGENIE — Step 1 (whole-genome model) + Step 2 (per-variant assoc)
regenie --step 1 \
    --bed ukb500k_geno \
    --phenoFile pheno.tsv --phenoCol T2D \
    --bsize 1000 --bt --out fit_step1

regenie --step 2 \
    --bed ukb500k_geno \
    --phenoFile pheno.tsv --phenoCol T2D \
    --pred fit_step1_pred.list \
    --bsize 400 --bt --firth --approx \
    --out assoc_T2D
