STEP 8 / 16

Joint Genotyping

gVCF → GenomicsDBImport → GenotypeGVCFs：N+1 可擴充的多樣本聯合分型策略。

gVCF → GenomicsDBImport → GenotypeGVCFs: the scalable N+1 strategy for multi-sample joint calling.

概覽

為什麼要做 Joint Genotyping？

單樣本 calling 在低深度位點容易把真實雜合子誤判為純合參考 (1/0 → 0/0)。Joint genotyping 把同 cohort 的多樣本資訊合併，讓某樣本「邊緣」的雜合證據能在其他樣本看見同位點變異時被重新評估——對 罕見等位基因、家系研究、族群參考面板尤其重要。

GATK Best Practices 採用兩階段架構：每個樣本先用 HaplotypeCaller -ERC GVCF 產生 gVCF（包含每個位點的證據，不只變異點），再由 GenotypeGVCFs 在 cohort 層級重新整合。優勢：新增樣本只要產 gVCF + 重跑 joint 階段，不必重新比對。

Single-sample calling tends to mistake true heterozygotes for homozygous reference (1/0 → 0/0) at low depth. Joint genotyping pools cohort information so weak heterozygous evidence in one sample is re-evaluated when the same site varies in others — essential for rare alleles, family studies, and population panels.

GATK Best Practices uses a two-stage design: each sample runs HaplotypeCaller -ERC GVCF producing a gVCF (evidence at every site, not just variants), then GenotypeGVCFs reintegrates them at cohort scale. Adding a new sample only requires its gVCF + a rerun of the joint stage — no realignment.

💡
核心觀念：gVCF 不是「只記變異」的 VCF，它對每個位點都記錄 reference confidence block（連續 ref 區段壓縮為 BLOCK 行）+ 候選變異 PL，是 N+1 流程的基石。 Key concept: a gVCF is not a "variant-only" VCF — every site is covered, with continuous reference stretches compressed into reference-confidence blocks plus candidate-variant PLs. It is the cornerstone of the N+1 design.
VCF 規格

一、必懂的 FORMAT/INFO 欄位

🧬

GT

基因型呼叫：0/0 ref-hom、0/1 het、1/1 alt-hom、1/2 雙 alt、./. missing。phased 用 |。

Genotype: 0/0 ref-hom, 0/1 het, 1/1 alt-hom, 1/2 bi-allelic, ./. missing. Phased uses |.

📊

AD / DP

AD = 每個 allele 的 read 數；DP = informative depth。AD[1]/DP 即 VAF，是 somatic 與 mosaicism 判讀關鍵。

AD = per-allele read count; DP = informative depth. AD[1]/DP = VAF, key for somatic & mosaicism reads.

🎯

GQ

Genotype Quality：第二好基因型與最佳的 PL 差，上限 99。GQ < 20 通常視為低信心。

Genotype Quality: PL gap between best and 2nd-best call, capped at 99. GQ < 20 = low confidence.

🔢

PL

三種基因型 (0/0, 0/1, 1/1) 的 Phred-scaled 機率，最佳者為 0。PL=[0, 30, 60] 代表強勢 ref。

Phred-scaled likelihoods of (0/0, 0/1, 1/1); best is 0. PL=[0,30,60] = strong ref.

📐

QUAL

位點層級的變異存在機率 (Phred)。joint 後常被 QD 取代為 filtering 依據。

Site-level Phred probability the variant exists. Often replaced by QD downstream for filtering.

🗂️

INFO/AC,AN,AF

AC = alt allele 計數、AN = 總 allele 數、AF = AC/AN。族群篩選 (rare variant) 的核心。

AC = alt allele count, AN = total alleles, AF = AC/AN. Core fields for rare-variant filtering.

程式碼

實作範例

# Step 1: per-sample gVCF
gatk HaplotypeCaller \
    -R Homo_sapiens.GRCh38.fa \
    -I bam/S1.bqsr.bam \
    -O gvcf/S1.g.vcf.gz \
    -ERC GVCF \
    -L intervals.bed \
    --tmp-dir tmp/
# Step 2: import all gVCFs to GenomicsDB (per-interval)
gatk GenomicsDBImport \
    --sample-name-map cohort.map \
    --genomicsdb-workspace-path gdb/chr1 \
    -L chr1 \
    --batch-size 50 \
    --reader-threads 4

# cohort.map (tsv): sample_idgvcf_path
# Adding new samples later — use update mode
gatk GenomicsDBImport --genomicsdb-update-workspace-path gdb/chr1 \
    --sample-name-map new_samples.map -L chr1
# Step 3: joint genotyping across cohort
gatk GenotypeGVCFs \
    -R Homo_sapiens.GRCh38.fa \
    -V gendb://gdb/chr1 \
    -O joint/cohort.chr1.vcf.gz \
    -L chr1 \
    --tmp-dir tmp/

# Merge per-chrom VCFs into cohort VCF
gatk GatherVcfs -I joint/cohort.chr1.vcf.gz ... -O joint/cohort.vcf.gz

1. gVCF 與一般 VCF 的根本差別是？

1. The fundamental difference between gVCF and a regular VCF?

A. gVCF 是二進位、VCF 是文字A. gVCF is binary, VCF is text
B. gVCF 只記變異點B. gVCF only records variants
C. gVCF 記錄每個位點的證據（含 reference confidence block），讓後續 joint 可重評估C. gVCF records evidence at every site (including reference-confidence blocks) so joint calling can re-evaluate
D. gVCF 不含 PL 欄位D. gVCF lacks PL fields

2. 為何聯合分型比單樣本 calling 更能偵測罕見雜合子？

2. Why does joint genotyping detect rare heterozygotes better than single-sample calling?

A. 因為 joint 提升每個樣本的覆蓋深度A. Joint increases per-sample depth
B. 因為同 cohort 其他樣本在該位點是變異時，能更新先驗機率，把邊緣證據救回B. Other cohort samples varying at the same site update the prior, rescuing borderline evidence
C. 因為 joint 會降低錯誤率C. Joint lowers raw sequencing error rate
D. 因為 joint 用更新的演算法D. Joint uses a newer algorithm

3. 是非題：N+1 問題下，GenomicsDBImport 比 CombineGVCFs 更適合大型 cohort 的增量更新。
