STEP 9 / 16

VQSR 與 Hard Filter

用 Gaussian mixture 或 CNN 從 cohort 篩出真變異——什麼時候用哪個？

Filter true variants from cohort using Gaussian mixtures or CNNs — when to use which?

概覽

變異過濾的兩條路線

Joint VCF 仍混雜大量假陽性 (PCR artefacts、mapping ambiguity、低複雜度區域)。過濾的目標是在不犧牲 sensitivity 的前提下移除 FP。GATK 提供兩條路徑：

路線 A — VQSR（Variant Quality Score Recalibration）：用 Gaussian mixture 模型，以 HapMap、1000G omni、dbSNP 等「known truth sites」為正例，學習 QD、MQ、FS、SOR、MQRankSum、ReadPosRankSum 等多維特徵的分佈，輸出 VQSLOD 分數。需要至少 30 個 exome 或 1 個 WGS 的資料量才能穩定收斂。

路線 B — Hard Filter：以固定閾值排除明顯異常的位點。用於 trio / 小 cohort 等樣本太少、VQSR 不穩定的情境。GATK 4 還新增了 CNNScoreVariants（1D 用序列、2D 用 pileup tensor），對單樣本可達接近 VQSR 的表現。

The joint VCF still contains many false positives (PCR artefacts, mapping ambiguity, low-complexity regions). Filtering aims to remove FPs without sacrificing sensitivity. GATK offers two paths:

Route A — VQSR: a Gaussian mixture over multi-dimensional features (QD, MQ, FS, SOR, MQRankSum, ReadPosRankSum), trained on truth sites (HapMap, 1000G omni, dbSNP) to output VQSLOD. Needs at least 30 exomes or 1 WGS for stable fitting.

Route B — Hard Filter: fixed thresholds, for trios / small cohorts where VQSR cannot converge. GATK 4 added CNNScoreVariants (1D on sequence, 2D on pileup tensors), reaching near-VQSR performance on a single sample.

💡
2026 趨勢：DRAGEN 4.x 內建 DRAGEN-ML 過濾器（深度學習，無需訓練 cohort），單樣本即可使用，已逐步取代 VQSR；GATK 也持續強化 CNN 路線。 2026 trend: DRAGEN 4.x ships DRAGEN-ML (deep learning, no cohort training needed) on single samples, displacing VQSR; GATK keeps strengthening the CNN route.

⚠️
FS vs SOR：FS 對深度敏感，深度高時容易誤判；SOR 對深度較穩。WGS 高深度位點建議以 SOR 為主、FS 為輔。 FS vs SOR: FS is depth-sensitive (false positives at high depth); SOR is more depth-robust. Prefer SOR (with FS as backup) at high-depth WGS sites.

Q1:
Cohort ≥ 30 exomes or ≥ 1 WGS? → Yes → VQSR (run SNV and Indel separately).
Q2:
Single sample / small trio? → Yes → CNNScoreVariants 2D (pileup), or Hard Filter fallback.
Q3:
Running DRAGEN 4.x? → Yes → Use DRAGEN-ML, skip VQSR.
Q4:
FP still high post-filter? → Yes → Apply blacklists (ENCODE, low-complexity, segmental duplication).

程式碼

實作範例

# SNV VQSR — 建模 + 套用
gatk VariantRecalibrator \
    -V joint/cohort.vcf.gz \
    --resource:hapmap,known=false,training=true,truth=true,prior=15 hapmap_3.3.hg38.vcf.gz \
    --resource:omni,known=false,training=true,truth=true,prior=12 1000G_omni2.5.hg38.vcf.gz \
    --resource:1000G,known=false,training=true,truth=false,prior=10 1000G_phase1.snps.high_confidence.hg38.vcf.gz \
    --resource:dbsnp,known=true,training=false,truth=false,prior=2 dbsnp.vcf.gz \
    -an QD -an FS -an MQ -an SOR -an MQRankSum -an ReadPosRankSum \
    -mode SNP --tranches-file snv.tranches \
    -O snv.recal --rscript-file snv.plots.R

gatk ApplyVQSR -V joint/cohort.vcf.gz --recal-file snv.recal \
    --tranches-file snv.tranches --truth-sensitivity-filter-level 99.0 \
    -mode SNP -O joint/cohort.snv-recal.vcf.gz
# Hard Filter — SNV
gatk VariantFiltration -V joint/cohort.snv.vcf.gz \
    --filter-expression "QD < 2.0" --filter-name "QD2" \
    --filter-expression "FS > 60.0" --filter-name "FS60" \
    --filter-expression "MQ < 40.0" --filter-name "MQ40" \
    --filter-expression "MQRankSum < -12.5" --filter-name "MQRS" \
    --filter-expression "ReadPosRankSum < -8.0" --filter-name "RPRS" \
    --filter-expression "SOR > 3.0" --filter-name "SOR3" \
    -O joint/cohort.snv.hardf.vcf.gz

# Indel
gatk VariantFiltration -V joint/cohort.indel.vcf.gz \
    --filter-expression "QD < 2.0 || FS > 200.0 || SOR > 10.0 || ReadPosRankSum < -20.0" \
    --filter-name "INDEL_BAD" -O joint/cohort.indel.hardf.vcf.gz
# CNNScoreVariants 2D — 單樣本深度學習過濾
gatk CNNScoreVariants \
    -V single/cohort.vcf.gz \
    -R Homo_sapiens.GRCh38.fa \
    -I bam/S1.bqsr.bam \
    --tensor-type read_tensor \
    -O cnn/cohort.cnn.vcf.gz

gatk FilterVariantTranches -V cnn/cohort.cnn.vcf.gz \
    --snp-tranche 99.95 --indel-tranche 99.4 \
    --resource hapmap_3.3.hg38.vcf.gz --resource mills_indels.vcf.gz \
    --info-key CNN_2D -O cnn/cohort.cnn-filt.vcf.gz
