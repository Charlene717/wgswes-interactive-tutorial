STEP 2 / 16

原始定序 QC (FASTQ)

在比對之前識破測序問題：低品質 base、adapter 污染、GC bias、duplication、過度 PCR。

Catch sequencing problems before alignment: low-Q bases, adapter contamination, GC bias, duplication, over-PCR.

概覽

為什麼 FASTQ QC 是必要的？

FASTQ 是所有下游分析的源頭。Garbage in, garbage out：若原始定序資料有 adapter 污染、過度 PCR、樣本污染或 base quality 嚴重下降，無論後面用多先進的 caller 都會出現大量假陽性。

QC 不只是「跑 FastQC 看綠色打勾」——你必須理解每個模組的失敗意義，特別是 WGS/WES 與 RNA-seq 的判讀差異 (例如 K-mer 重複在 WGS 是 PCR over-duplication；在 RNA-seq 則可能是高表達基因)。

FASTQ is the source of all downstream analyses. Garbage in, garbage out: if raw reads have adapter contamination, PCR over-duplication, sample swap, or quality drop, even the best caller will produce false positives.

QC is not just "run FastQC and check for green ticks" — you must understand what each module's failure means, and how WGS/WES differs from RNA-seq (e.g., k-mer duplication in WGS suggests PCR over-cycling; in RNA-seq it may just be a highly expressed gene).

💡
核心工具鏈：FastQC (Andrews 2010, Babraham) 每樣本獨立報告 → MultiQC (Ewels 2016, Bioinformatics) 多樣本彙整 → fastp (Chen 2018, Bioinformatics) 一步完成 QC+trimming，已成現代主流。 Core toolchain: FastQC (Andrews 2010, Babraham) per-sample → MultiQC (Ewels 2016, Bioinformatics) aggregation → fastp (Chen 2018, Bioinformatics) one-step QC+trim — the modern default.
核心指標

一、FastQC 11 個模組必懂解讀

📊

Per base sequence quality

Q score 沿讀長分佈。Illumina Q30 = 99.9% 準確；3' 末端常因化學疲乏下降。失敗：箱形圖中位數 < Q20。

Q distribution along read. Q30 = 99.9% accuracy. 3' end usually drops due to chemistry fatigue. Fail: median < Q20.

🧬

Per sequence GC content

人類 ~41% GC。雙峰常代表污染 (其他物種、adapter dimer)。WGS 期望單峰，WES 因捕獲探針可能略偏。

Human ~41% GC. Bimodal often = contamination (another species, adapter dimers). WGS expects unimodal; WES may be slightly shifted by capture probes.

🔁

Sequence duplication

WGS 高 dup (>30%) 暗示 PCR over-cycling 或低 input；WES 可達 30–50%（高深度本就重複多），需 MarkDup 後再判斷。

High dup (>30%) in WGS suggests over-PCR or low input. WES often 30–50% (high depth inflates duplication) — judge after MarkDup.

🪝

Adapter content

3' 末端 adapter 表示插入片段過短。Illumina TruSeq、Nextera、xGen 各有特定序列。fastp 可自動偵測。

3' adapter = insert too short. Illumina TruSeq, Nextera, xGen each have distinct sequences. fastp auto-detects.

🔬

Overrepresented seqs

WGS 出現特定 reads 多次 → 警示 PCR 偏差、rRNA 殘留、index hopping (NovaSeq patterned flowcell)。

WGS reads recurring → flag PCR bias, rRNA residue, or index hopping (NovaSeq patterned flowcell).

🎲

Per tile sequence quality

NovaSeq / NextSeq 顯示 flowcell 區域品質。一片 lane 紅色暗示氣泡、汙染、scratch。

Shows per-tile quality on NovaSeq/NextSeq. A red band = bubble, contamination, or scratch.

互動模擬

Phred Q score → base accuracy

拖動 Q score 觀察錯誤率變化。Q = -10·log₁₀(P_error)。理解 Q20/Q30/Q40 對應的實際錯誤率，才能評估 reads 是否值得保留。

Drag Q to see error rate. Q = -10·log₁₀(P_error). Understanding Q20/Q30/Q40 in absolute terms helps decide whether reads are worth keeping.

決策引導

二、QC 失敗時的決策樹

🌳 QC 異常排查

Q1:
Per base quality 3' 端嚴重下降？ → 是 → 用 fastp/Trimmomatic quality trim，閾值 Q15-Q20。不要 over-trim。
Q2:
Adapter content > 5%？ → 是 → 啟用 adapter trimming。fastp 預設可自動偵測。
Q3:
GC distribution 雙峰？ → 是 → 跑 FastQ Screen / Kraken2 檢查物種污染。
Q4:
Duplication > 40%（WGS）？ → 是 → 檢查 input DNA 量、PCR cycles。MarkDup 後若 dup > 50% 仍嚴重，考慮重做。
Q1:
3' end quality drop? → Yes → Quality trim with fastp/Trimmomatic at Q15–Q20. Avoid over-trimming.
Q2:
Adapter content > 5%? → Yes → Enable adapter trimming; fastp auto-detects by default.
Q3:
Bimodal GC distribution? → Yes → Run FastQ Screen / Kraken2 for species contamination.
Q4:
Duplication > 40% (WGS)? → Yes → Check input DNA mass and PCR cycles. If > 50% after MarkDup, consider re-prep.
⚠️
常見誤解：FastQC 給「紅色」未必代表資料壞——只代表「不符合通用 mRNA-seq 假設」。WGS/WES/amplicon 的判讀標準完全不同。一定要結合定序模式與生物背景判讀。 Common misconception: FastQC "red" doesn't always mean bad data — it means "doesn't match generic mRNA-seq assumptions". WGS/WES/amplicon have different expectations. Always interpret in context.
程式碼

QC 工具實作

# FastQC + MultiQC for raw FASTQ QC
fastqc -t 8 -o qc/raw fq/*.fq.gz

# Aggregate multi-sample reports
multiqc qc/raw -o qc/multiqc_raw

# Optional: species contamination check
fastq_screen --conf fastq_screen.conf fq/sample_R1.fq.gz
# fastp: one-step QC + trimming (recommended)
fastp \
    -i fq/S1_R1.fq.gz -I fq/S1_R2.fq.gz \
    -o trim/S1_R1.fq.gz -O trim/S1_R2.fq.gz \
    -h qc/fastp/S1.html -j qc/fastp/S1.json \
    --detect_adapter_for_pe \
    --qualified_quality_phred 15 \
    --length_required 50 \
    --thread 8

# Then aggregate
multiqc qc/fastp -o qc/multiqc_fastp
