CNV / SV 偵測

超越 SNV / Indel：缺失、重複、倒置、插入、易位——用 split read、discordant pair 與 read depth 三大訊號偵測。

Beyond SNV/Indel: DEL, DUP, INV, INS, TRA — detected via split-read, discordant-pair, and read-depth signals.

概覽

什麼是 SV？什麼是 CNV？

SV (Structural Variant) = 大於 50 bp 的結構變異，包含 5 大類型：DEL（缺失）、DUP（重複）、INV（倒置）、INS（插入）、TRA / BND（易位 / breakend）。

CNV (Copy Number Variant) 是 SV 的子集，特指影響拷貝數的變異（DEL + DUP）。INV 不改變拷貝數所以不算 CNV。臨床上 CNV 涵蓋 chr 21 trisomy、22q11.2 deletion 等知名疾病。

SV 偵測比 SNV 困難得多：訊號分散、breakpoint 不一定精確、repetitive 區域容易假陽性。T2T-CHM13 參考（Nurk 2022 Science、Aganezov 2022 Science）讓 ~5% 額外 SV 得以解析，特別是 centromere 與 acrocentric 區域。

SV (Structural Variant) = variation > 50 bp, five types: DEL, DUP, INV, INS, TRA / BND (translocation / breakend).

CNV (Copy Number Variant) is a subset of SV — only DEL + DUP that change copy number. INV doesn't change copy number, so it isn't a CNV. Clinically, CNVs include chr 21 trisomy and 22q11.2 deletion.

SV detection is harder than SNV: signals are diffuse, breakpoints imprecise, and repeats produce FPs. T2T-CHM13 (Nurk 2022, Aganezov 2022 Science) resolves ~5% more SVs per genome, especially in centromeric/acrocentric regions.

💡
三大訊號：(1) Split-read：單一 read 跨越 breakpoint 被切兩半；(2) Discordant pair：PE 兩端距離 / 方向異常；(3) Read depth：區段覆蓋深度偏離預期。專業 caller 同時用 2-3 種訊號。 Three signals: (1) Split-read: a read crosses a breakpoint and is soft-clipped; (2) Discordant pair: PE distance/orientation abnormal; (3) Read depth: regional coverage deviates from expectation. Production callers use 2–3 signals jointly.

Q2:
WES？ → SV 用 Manta exome 模式；CNV 用 CNVkit（targeted）或 XHMM/CoNIFER（cohort）。

Q4:

Q1:
WGS short-read germline? → Manta + DELLY (consensus), add GATK gCNV for CNV.
Q2:
WES? → SV via Manta exome mode; CNV via CNVkit (targeted) or XHMM/CoNIFER (cohort).
Q3:
Long-read (PacBio HiFi / ONT)? → Sniffles2 primary + cuteSV for recall.
Q4:
Tumor / matched normal? → Manta + GRIDSS2, cross-check with SvABA.
⚠️
WES CNV 警告：WES 因捕獲偏差使 read-depth 信號雜訊大，CNV 解析力遠不如 WGS。臨床確認需要 array-CGH 或 MLPA。GATK gCNV 需要 ≥ 100 個技術相似 (kit + lab) 樣本建立 panel-of-normals。 WES CNV caveat: capture bias makes read-depth noisy — WES CNV resolution is far worse than WGS. Clinical confirmation needs aCGH or MLPA. GATK gCNV requires ≥ 100 technically similar (kit + lab) samples to build a panel-of-normals.

程式碼

實作範例

# Manta — germline SV
configManta.py \
    --bam bam/S1.bqsr.bam \
    --referenceFasta Homo_sapiens.GRCh38.fa \
    --runDir manta/S1
manta/S1/runWorkflow.py -m local -j 8

# 結果：manta/S1/results/variants/diploidSV.vcf.gz
# 過濾 PASS + 大於 100 bp + 非低複雜度區
bcftools view -f PASS manta/S1/results/variants/diploidSV.vcf.gz \
  | bcftools view -i 'INFO/SVLEN<-100 || INFO/SVLEN>100' \
  > sv/S1.manta.pass.vcf
# CNVkit — WES somatic CNV
cnvkit.py batch \
    bam/tumor.bam \
    --normal bam/normal*.bam \
    --targets target.bed \
    --annotate refFlat.txt \
    --fasta Homo_sapiens.GRCh38.fa \
    --access access-5kb-mappable.hg38.bed \
    --output-reference cnvkit/pon.cnn \
    --output-dir cnvkit/ \
    --processes 8

cnvkit.py scatter cnvkit/tumor.cnr -s cnvkit/tumor.cns -o cnvkit/tumor.pdf
cnvkit.py call cnvkit/tumor.cns -o cnvkit/tumor.call.cns
# GATK gCNV — 大 cohort germline CNV
gatk PreprocessIntervals -L target.bed --interval-merging-rule OVERLAPPING_ONLY \
    --bin-length 0 -R ref.fa -O intervals.preprocessed.interval_list

gatk CollectReadCounts -I bam/S1.bqsr.bam -L intervals.preprocessed.interval_list \
    --interval-merging-rule OVERLAPPING_ONLY -O counts/S1.counts.tsv

# cohort PON (建議 ≥ 100 同技術背景樣本)
gatk GermlineCNVCaller --run-mode COHORT \
    -L intervals.preprocessed.interval_list \
    -I counts/*.counts.tsv \
    --contig-ploidy-calls ploidy/calls \
    --output gcnv/ --output-prefix cohort
